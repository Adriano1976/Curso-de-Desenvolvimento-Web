1. Quais as vantagens e desvantagens da topologia física de barramento?

As vantagens da topologia de barramento são: Implementação e manutenção simples, menor quantidade de cabos de interligação portanto é mais barata e facilidade de ampliar, tanto na quantidade de dispositivos, quanto na extensão. As desvantagens são: taxa de transmissão pode sofrer diminuição caso existam muitos computadores ligados ao barramento e uma problema no barramento pode parar a rede toda. Neste vídeo, feito e disponibilizado no canal Espiral, explica o que é a topologia de barramento e como funciona. Durante a explicação, você verá as vantagens e desvantagens dessa topologia.


2. Quais as vantagens e desvantagens da topologia física em estrela?

As vantagens da topologia em estrela são: facilidade de expansão (basta conectar ao hub) e maior confiabilidade (o hub não permite que a falha de um equipamento pare a rede). As desvantagens são: em caso de falha no hub a rede para e custo maior de instalação (equipamento extra). Neste vídeo, feito e disponibilizado no canal Espiral, explica o que é a topologia em estrela e como funciona a sua estrutura, qual a sua característica. Observem as vantagens e desvantagens dessa topologia.


3. Quais as vantagens e desvantagens da topologia física em anel?

A principal vantagem associada a essa topologia é que ela permite (associada a topologia lógica Token Ring) taxas de transmissão maiores. As desvantagens são: ocorrendo falha em um dispositivo a rede pode parar, para ampliar a rede é necessário parar o seu funcionamento e o custo é um pouco elevado (mais cabos e dispositivos). Neste vídeo, feito e disponibilizado no canal Espiral, explica o que é a topologia em anel e como funciona a sua estrutura, qual a sua característica e observe a vantagem e desvantagem dessa topologia.

5. Os repetidores atuam em que camada do modelo OSI? Por quê?

Eles atuam na camada Física do modelo OSI, pois apenas repetem os pacotes de dados que recebem, não realizando qualquer tipo de tratamento nesses dados. Neste vídeo, feito e disponibilizado no canal Espiral, explica quais as camadas físicas do modelo OSI e como funciona essas camadas físicas.


6. Como as pontes podem ser usadas para segmentar uma LAN?

As pontes “observam” os dados que chegam até ela, para verificar o seu destino. Se o destino for dentro da própria rede que originou o pacote de dados, ela não permite que os dados trafeguem em outro segmento da rede. Dessa forma, os dados de um segmento só são enviados para outro segmento quando o destino for naquele segmento, caso contrário ficam confinados ao segmento que os originou.

7. Qual a vantagem que os switches apresentam sobre as pontes?

Uma ponte só permite a conexão de dois segmentos a ela, o que exige uma quantidade maior de pontes para interligar vários segmentos. O Switch permite que vários segmentos sejam conectados em somente um equipamento, apresentando as mesmas funcionalidades da ponte. Neste vídeo, feito e disponibilizado no canal Espiral, explica o que é um switch e como é o seu funcionamento.


8. Como funciona o princípio do CSMA CD em uma LAN Ethernet?

Os computadores devem aguardar um momento em que o segmento esteja sem atividades para enviar os seus dados (os computadores "ouvem" o segmento). Caso nenhum outro computador tente enviar dados durante o período em que os dados estão trafegando, eles chegarão ao destino sem maiores problemas. Se outro computador transmitir dados exatamente no mesmo momento de transmissão do primeiro, ocorrerá a colisão, então, os computadores que tentaram transmitir interrompem a transmissão, aguardam um tempo aleatório, mas diferente para cada computador (da ordem de milissegundos) e tentam retransmitir. Essa diferença de tempos de espera é uma tentativa de diminuir a possibilidade de nova colisão. No entanto, podem ocorrer colisões com dados de outros computadores. Nesse caso, o processo de espera é novamente iniciado e, assim, sucessivamente, enquanto houver necessidade de enviar dados pelo meio de comunicação. Neste vídeo, disponibilizado por Andre Peres, em seu canal, explica com detalhes como funciona o CSMA CD.


9. Como uma LAN Token Ring procura eliminar o problemas das colisões?

Enquanto nenhum computador tiver dados a transmitir (rede sem atividades) estará circulando pela rede (em uma topologia física em anel) um pacote de dados especial conhecido como token. Esse pacote é enviado de dispositivo para dispositivo, que pega o pacote, lê o seu conteúdo e, se não tiver nada a fazer com ele, o deixa circulando. Quando um computador necessitar transmitir dados, ele deve esperar a passagem do token livre, então, anexa os seus dados ao token e envia para o próximo computador do anel, este recebe o token e verifica se os dados são endereçados a ele. Caso não sejam, ele envia novamente o token pelo anel até que ele chegue ao dispositivo de destino dos dados. Nesse ponto, o dispositivo pega os dados e libera o token para que outro computador possa enviar seus dados. Dessa forma, não existe colisão, mas poderia ocorrer outro problema: o mesmo dispositivo sempre consegue pegar o token no anel antes ao computador seguinte, mas, fazendo isso, monopoliza a transmissão. Para resolver esse problema, a topologia Token Ring não permite que o mesmo dispositivo pegue o token novamente, logo após ter acabado de enviar dados. Ele deve esperar pela próxima "volta" do token. Neste vídeo ilustrativo mostra como funciona uma LAN Token Ring e como essa arquitetura elimina o problema da colisões entre os pacotes de dado.
