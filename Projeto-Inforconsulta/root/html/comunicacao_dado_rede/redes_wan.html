Redes Wan

1. Quais são as principais aplicações de uma WAN?

Permitir que um usuário remoto acesse os dados que estão nos computadores da empresa, permitir que filiais espalhadas geograficamente, ou em regiões remotas, possam utilizar os recursos disponíveis no computador central da matriz ou permitir que LANs espalhadas geograficamente possam ser agrupadas em uma única rede de grande alcance geográfico. Nesta vídeo-aula, feito e disponibilizado pelo canal Q-Cursos Network, além de falar o que é a rede WAN, como funciona, ainda explica qual a diferença entre redes WAN e LAN.


2. A internet pode ser considerada uma WAN? Justifique.

De certa forma sim, pois ela é uma rede que interliga computadores por todo o mundo, não pertencendo a nenhuma empresa específica.

3.Qual a função de um roteador em redes WANs?

O roteador tem a função de encaminhar pacotes de dados no ambiente de sub-redes de uma WAN para ele chegue ao seu destino.

4. Quais são os protocolos de roteamento mais conhecidos e como eles funciona.

Os protocolos de roteamento mais conhecidos são:
  Shortest Path: é um método estatístico que o router utiliza para montar um grafo de rede (esquema que indica quem está conectado a quem) e o utiliza para encontrar a menor rota em direção ao destino do pacote. Para descobrir essa menor rota, além do grafo, ele utiliza o número de roteadores "saltados" e a distância.
  Flooding: é um método estatístico no qual o pacote que chega ao router é enviado a todas as suas saídas (menos para aquela de onde ele veio). Um exemplo de utilização desse método é um ambiente de banco de dados distribuídos em que todos os bancos de dados devem ser atualizados com o mesmo dado.
  Flow-Based: é um método estatístico no qual cada ligação entre dois pontos da rede são levados em consideração. A partir desse conhecimento, é aplicada uma métrica para determinar a rota mais conveniente. Essa métrica é predeterminada.
  Distance Vector: esse método dinâmico é um dos mais utilizados no roteamento. O router mantém uma tabela com as menores distâncias para cada destino possível. Essa tabela é mantida e atualizada por meio da troca dinâmica de informações entre routers adjacentes (que possuem conexões uns com os outros).
  Link State: esse método dinâmico também é bastante utilizado e se baseia em:
    1. Descobrir outros routers que sejam seus vizinhos e quais são os seus endereços na rede;
    2. Calcular o retardo e “custo” (REVISADO) da transmissão de um pacote de dados para cada vizinho;
    3. Construir o pacote de acordo com o que aprendeu sobre os seus vizinhos (routers);
    4. Enviar o pacote que possui informações sobre todos os roteadores;
    5. Armazenar as informações dessa última transmissão do pacote (sobre as rotas) para uso futuro. Neste vídeo, disponibilizado feito pelo NIC.br. e disponibilizado no canal NICbrvideos, explica que os mecanismos de roteamento e encaminhamento atuam como um GPS, fornecendo um mapa para que os dados cheguem ao seu destino.


5. Para que servem os gateways?

O gateway é um roteador especial que conduz pacotes de redes completamente diferentes (arquiteturas diferentes) e podem ser entendidos como conversores de protocolo.

6. O que são VPNs? Por que elas são criadas?

VPN é uma rede que se sobrepõe a redes públicas (como a Internet), criando um canal virtual de comunicação entre localidades, como se fosse uma linha privativa. Elas foram criadas para tentar resolver o problema que as empresas possuíam de transferir dados entre localidades distantes por linhas privativas, que são muito caras.
