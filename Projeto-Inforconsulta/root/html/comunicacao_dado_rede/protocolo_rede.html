Protocolo de Rede

1. Defina protocolo. Na sua opinião é um software, um hardware ou uma mistura dos dois.

Protocolo é um conjunto de regras que controla a comunicação entre dois computadores para que ela ocorra de forma eficiente e sem erros, que devem ser detectados e tratados. Na verdade o protocolo é uma mistura dos 2, hardware e software. Nesta vídeo-aula, disponibilizado pelo canal Espiral, explica o que é um protocolo de rede e como é o seu funcionamento.


2. O que é modelo ISO/OSI e por que foi criado?

É um modelo teórico de referência que serve para explicar como deve funcionar uma rede de computadores, portanto servindo como modelo para a criação de novos protocolos. Nesta vídeo-aula, feito por Paulo Kretcheu, explica sobre o modelo OSI, modelo de referência para interconexão de dispositivos de rede.


3. Como uma camada do modelo OSI no equipamento transmissor pode se comunicar com a camada correspondente no equipamento receptor?

Cada camada no modelo OSI acrescenta um cabeçalho aos dados com informações que só podem ser lidos pela camada correspondente no destino desses dados (receptor). Nesta vídeo-aula disponibilizado pela Espiral, explica sobre as camadas do modelo OSI e suas aplicações.


4. Qual a diferença entre um sistema orientado à conexão e um sistema sem conexão?

Em um sistema orientado à conexão, antes que a transmissão ocorre é necessário estabelecer uma conexão entre origem e destino, que deve estar ativa enquanto os dados são transmitidos. Essa conexão só é desfeita quando a transmissão se encerra. Já um sistema sem conexão, envia os pacotes de dados, sem garantia alguma que chegarão ao destino, pois não é estabelecida uma conexão. Os pacotes de dados podem trafegar por caminhos diferentes para chegar ao destino.

5. Explique as funções das camadas do modelo OSI.

Camada de Aplicação – responsável por permitir que as aplicações do computador possam acessar a rede. Camada de Apresentação - responsável por tratar o modo como os diferentes sistemas representam os dados, realizando isso por meio de uma formatação dos dados e de uma seleção da sintaxe adequada. Camada de Sessão – responsável por estabelecer, gerenciar e terminar uma conexão entre os diferentes computadores (uma sessão). Camada de Transporte – responsável por garantir a transferência de dados, de forma confiável, entre origem e destino, independente de qual rede física é utilizada no momento. Camada de Rede – responsável pela operação das sub-redes, se preocupa com o envio de pacotes da origem ao destino e do roteamento desses pacotes. Camada de Enlace – responsável em transformar o canal de transmissão em um canal livre de erros. Cuida das regras de intercâmbio de dados entre dois pontos. Camada Física – responsável pelos detalhes físicos de transmissão de dados. Neste vídeo explicativo, disponibilizado na canal de Willian iDELfonso, fala com detalhes as funções das camadas do modelo OSI


6. Por que a camada de transporte é considerada a mais importante do modelo OSI?

Porque ela é responsável em garantir uma transferência de dados de modo confiável. Além disso, ela é a camada que liga as camadas superiores, que tratam da comunicação entre origem e destino (não se importando por quantos computadores esses dados devem passar nesse processo) e as camadas inferiores, que tratam da comunicação entre dois equipamentos interligados diretamente. Nesta vídeo-aula, disponibilizado pelo canal Espiral, explica qual a idéia da camada de transporte e como funciona.


7. Como funciona a comutação de pacotes?

As mensagens são transmitidas em blocos de tamanho fixo, portanto os dados são “quebrados” em pacotes de dados menores. O roteador é responsável por receber o pacote, analisar para onde ele vai e escolher o melhor caminho para ele. Dessa forma, não existe garantia de que todos os pacotes vão percorrer o mesmo caminho para atingir o destino. Neste vídeo explicativo, disponibilizado pelo Centro de Formação Profissional Juiz de Fora mostra como funciona a comutação de pacotes.


8. Por qual motivo não é suficiente o número de endereços gerados pelo endereçamento IPv4 (TCP/IP)?

Porque as empresas (ou pessoas) que necessitam adquirir endereços IP para a sua rede, devem adquirir uma classe inteira, o que normalmente, é muito mais do que ele necessita para a sua operação. Dessa forma boa parte dos endereços acaba ficando sem utilização, mas ligados a quem os adquiriu. Assim, os endereços IP estão acabando. Neste vídeo explicativo, feito pelo canal de televisão Olhar Digital e disponibilizado no canal Tecnologia na rede, explica da mudança do IPv4 para o IPv6 e o que significa esta mudança.


9. Como o IPv6 se propõe a acabar com esse problema?

O IPv4 utiliza quatro números, para endereçar, que variam de 0 a 255, o que leva a existência de 4.228.250.625 endereços possíveis. O IPv6 propõe a utilização de 8 números, variando, cada um deles, de 0 a 65.535, o que leva a existência de 3,4 x 10 elevado á 38 endereços possíveis. Neste vídeo explicativo, feito pelo canal de televisão Olhar Digital e disponibilizado no canal OzirisBlack, explica o que é IPv6 e como funciona.


10. Quais as vantagens da tecnologia de transmissão ATM?

A principal vantagem da tecnologia ATM é a alocação dinâmica de banda, ou seja, a banda é alocada para cada tipo de informação que deverá trafegar, sendo alterada de acordo com a situação atual. Neste video elaborado para o materia sobre os Fundamentos de Telecomunicações, ministrada pelo profesor: Salvador Martínez Pagola, docente do Instituto Tecnológico de Pachuca, disponibilizado na canal Pako Ronq, explica sobre a transmissão ATM.
