Memorias

1. Quais são as principais características das memórias ROM?

Persistência (dados são mantidos mesmo sem alimentação de energia) e Segurança (modificação dos dados armazenados é difícil). Nesta vídeo-aula dada por Adriano Oliveira e disponibilizado por Bóson Treinamento, ensina como funciona e quais as características das memórias ROM.

2. Qual a principal diferença entre uma EEPROM e um Flash-ROM?

A diferença principal entre as duas é que nas memórias EEPROM é possível apagar o conteúdo de somente um endereço, enquanto nas memórias Flash-ROM todo o conteúdo é obrigatoriamente apagado. Nesta vídeo-aula, disponibi-lizada por Jose Willian, explica sobre as suas tecnologia e funcionamento no PC.

3. Compare as características das memórias RAM dinâmicas e estáticas.

As memórias RAM estáticas, utilizam mais componentes para armazenar os dados, com isso são mais caras e apresentam um tamanho maior, mas esses componentes permitem que o acesso aos dados seja mais rápido. As memórias RAM dinâmicas utilizam menos componentes para armazenar os dados, com isso são mais baratas e apresentam um tamanho menor, mas com isso, existe a necessidade da operação de refresh, o que as torna mais lentas que as memórias RAM estáticas. Nesta 2 vídeos-aulas, disponibilizados no canal ComoTudoFunciona, explica as diferenças entre memórias RAM e dinâmicas e estáticas.

4. Por que os sistemas de computação utilizam as memórias RAM dinâmicas como memórias principais, se eles são mais lentas?

Principalmente devido ao seu preço ser inferior, se comparado com as memórias RAM estáticas.


5. Uma memória apresenta temporização de 6-1-1-1 (modo burst). Caso essa memória não utilizasse o modo burst, quantos ciclos de clock seriam necessários para acessá-la?

Se essa memória não utilizasse o modo burst de acesso, ela gastaria o mesmo número de ciclos de clock para cada acesso aos dados, portanto a sua temporização seria 6-6-6-6, portanto seriam necessários 24 ciclos de clock para que o acesso acontecesse.


6. O que é latência de memória?

Quando a memória utiliza o modo burst para acessar os dados, sempre o pri-meiro valor da temporização é maior que os outros. Essa quantidade de ciclos de clock, maior que os outros é conhecido como latência de memória. Nesta vídeo-aula dada por Adriano Oliveira e disponibilizado por Bóson Treinamento, ensina sobre a latência e velocidade da memória.


7. Por que os sistemas de vídeo necessitam de memórias RAM especiais?

Porque para acessar dados referentes ao vídeo é necessária uma velocidade de acesso mais alta que aquelas obtidas pelas memórias utilizadas como memória principal. Neste vídeo, produzido pela emissora de TV Olhar Digital, explica como funciona as placas de vídeos com suas memórias RAM especiais e suas aplicações no mercado de consumo.

8. Aumentar a quantidade de memória principal aumenta a velocidade de processamento geral do sistema de computador? Explique.

A quantidade de memória principal em um sistema de computação afeta o seu desempenho, apesar de não aumentar a sua velocidade de processamento nem a velocidade de acesso aos dados. Aumentar a quantidade de memória instalada no sistema significa que o computador pode utilizar programas maiores ou acessar arquivos de dados processados maiores. Além disso, maior quantidade de memória faz o sistema processar os dados mais rapidamente, pois, apesar de não ser necessário copiar todo o programa para a memória principal para executá-lo (apenas a parte que for utilizada no momento), quanto mais couber, mais depressa será a sua execução.


9. Em que situação o método de detecção de erros ECC apresenta vantagens sobre o método de paridade?

Quando o barramento de dados do sistema de computação possuir pelo menos 64 bits de tamanho, pois nesse caso, tanto no método de paridade quanto no ECC serão utilizados 8 bits para detectar o erro, mas aí o ECC leva vantagem, pois ele consegue, além de detectar, corrigir os erros encontrados. Nesta vídeo-aula dada por Adriano Oliveira e disponibilizado por Bóson Treinamento, ensina sobre a detecção e correção de erros na memória RAM.


10. Um conjunto de bits, com paridade par, chega até o processador desse forma: 00001001 com bit de paridade igual a 1. Os dados que chegaram apresentam erro?

Sim, pois a quantidade total de bits iguais a 1 não é par (somando os bits iguais a 1 do conteúdo e do bit de paridade, encontra-se o valor 3).
